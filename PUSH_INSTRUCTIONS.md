# Как правильно пушить в репозиторий

## По шагам

1. **`git add .`** — добавить все изменения (или указать конкретные файлы).
2. **`git commit -m "описание изменений"`** — зафиксировать коммит с сообщением.
3. **`git push origin main`** — отправить коммиты в GitHub.

После пуша в `main` Railway сам подхватывает изменения и пересобирает проект.

## Важно

- Всегда пушить в ветку **main**.
- Remote должен быть **origin** (репозиторий `brianweb3/worms-arena`).
- Не коммитить `node_modules`, `dist`, `store.json` и лишние артефакты (должны быть в `.gitignore`).
